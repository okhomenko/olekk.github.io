# Content Strategy

The strongest positioning for this site is not a generic personal blog.

It should be a public body of work around:

- engineering systems
- AI-enabled software development
- product architecture
- technical leadership
- endurance running as a long-term performance system

## Core sections

- Engineering
- AI
- Systems
- Running
- Personal notes

## Internal linking rule

Every post should link to at least two related posts or section pages.

## Metadata rule

Every post should have:

- title
- description
- published date
- updated date when changed
- canonical URL
- topic/category
- tags

## Search intent

Prefer essays that answer durable questions:

- how engineers adapt to AI
- how teams keep code quality high
- how architecture creates leverage
- how product systems compound
- how endurance training maps to long-term work
